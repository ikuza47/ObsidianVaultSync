> [!SUMMARY] Кратко
> **Определение**: Создание контента алгоритмически, а не вручную
> **Применение**: Уровни, лут, враги, события

---

**Процедурная генерация** — метод создания контента с помощью алгоритмов, а не вручную. Позволяет создавать большое количество уникального контента с минимальными затратами ресурсов.

---

## Типы генерации

| Тип | Описание |
|-----|----------|
| **Генерация уровней** | Создание структуры уровня |
| **Генерация лута** | Распределение предметов |
| **Генерация врагов** | Размещение противников |
| **Генерация событий** | Случайные события |

---

## Алгоритмы генерации уровней

### Room-first (сначала комнаты)

Создание случайных комнат с последующим соединением коридорами.

**Преимущества:** Контроль над размером комнат, лёгкая балансировка.
**Недостатки:** Менее органичная форма.

### Cellular Automata (клеточные автоматы)

Использование правил сглаживания для создания пещер из случайного шума.

**Преимущества:** Органичные пещеры, естественный вид.
**Недостатки:** Меньше контроля, может создать изолированные зоны.

### BSP (Binary Space Partitioning)

Рекурсивное разделение пространства с созданием комнат в конечных секциях.

**Преимущества:** Равномерное распределение, гарантия связности.
**Недостатки:** Прямоугольная структура, предсказуемость.

### Drunkard's Walk (Пьяная прогулка)

Случайное блуждание по сетке с выкапыванием пути.

**Преимущества:** Органичные пещеры, гарантия связности.
**Недостатки:** Может выйти за границы, неравномерная плотность.

---

## Сиды (Seeds)

**Сид** — начальное значение для генератора случайных чисел. Одинаковый сид = одинаковый результат генерации.

**Применение:**
- Спидраны (все бегут один уровень)
- Шеринг («Зацени сид!»)
- Дебаг (воспроизведение багов)
- Daily runs (один уровень в день для всех)

## Проблемы генерации

| Проблема | Описание | Решение |
|----------|----------|---------|
| **Несвязные зоны** | Изолированные области | Проверка связности |
| **Непроходимые уровни** | Нет пути к выходу | Алгоритмы проверки проходимости |
| **Несбалансированный лут** | Слишком много/мало | Динамическая балансировка |

---

## Динамическая балансировка

### Pity Timer (Таймер жалости)

Если игроку не везёт долгое время — повысить шансы на удачу.

### Адаптивная сложность

Подстройка сложности под навыки игрока на основе его успехов/неудач.
