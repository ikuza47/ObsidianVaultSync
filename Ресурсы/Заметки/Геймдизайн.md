> **Определение**: Дизайн игровых механик и систем
> **Цель**: Создание увлекательного опыта

---

**Геймдизайн** — процесс проектирования правил, механик, систем и контента игры с целью создания увлекательного игрового опыта.

---

## MDA Framework

| Компонент | Описание |
|-----------|----------|
| **Mechanics** | Базовые правила и системы |
| **Dynamics** | Взаимодействие игрока с механикой |
| **Aesthetics** | Эмоциональный отклик игрока |

---

## Типы механик

| Тип | Описание |
|-----|----------|
| **Движение** | Перемещение по миру |
| **Бой** | Взаимодействие с врагами |
| **Прогрессия** | Развитие персонажа |
| **Экономика** | Ресурсы и торговля |
| **Исследование** | Познание мира |

---

## Баланс игры

### Типы баланса

| Тип | Описание |
|-----|----------|
| **Числовой** | Баланс значений (урон, HP, время) |
| **Тактический** | Разные стратегии равны по эффективности |
| **Прогрессии** | Сложность растёт со временем |
| **RNG** | Баланс удачи и навыка |

---

## Flow (Поток)

Состояние полной вовлечённости, достигаемое при балансе между вызовом и навыком игрока.

```
Вызов
  ▲
  │  Тревога (слишком сложно)
  │ ╱
  │╱───────┌──────────────┐
  │        │    ПОТОК     │
  │        │  (баланс)    │
  │        └──────────────┘
  │╲───────┘
  │ ╲
  │  ╲────── Скука (слишком легко)
  │
  └─────────────────────▶ Навык
```
